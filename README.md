# Eatensity — Feedback & Feature Requests

Welcome! This is the **public feedback tracker** for [Eatensity](https://eatensity.com) — India's food intelligence platform.

Use this repo to:

- 🐛 **Report bugs** you encounter in the web or mobile app
- 💡 **Request features** you'd like to see
- 🗣️ **Share suggestions** for improving the experience
- 🔍 **Browse existing feedback** and upvote what matters to you

## How to submit feedback

1. **Search first** — Check [existing issues](../../issues) to see if someone already reported the same thing. If so, add a 👍 reaction and/or comment with additional details.
2. **Create a new issue** — Click [New Issue](../../issues/new/choose) and pick the right template (Bug Report, Feature Request, or Suggestion).
3. **Fill in the details** — The more context you provide, the faster we can act.

## What happens next?

| Label | Meaning |
|-------|---------|
| `acknowledged` | We've seen it and added it to our backlog |
| `in-progress` | Actively being worked on |
| `resolved` | Fixed or shipped — check the linked release |
| `wont-fix` | Not planned (we'll explain why) |
| `duplicate` | Already tracked in another issue |

You'll receive **GitHub notifications** whenever we reply or update the status of your issue.

## Guidelines

- Be respectful and constructive
- One issue per report — don't bundle multiple bugs in a single issue
- Include device, OS, browser, and app version when reporting bugs
- Screenshots and screen recordings are very helpful
- Don't share personal health data or sensitive information in issues
